# Feature Spec: 휴일 관리 (Holiday Management)

## 목적
워크스페이스(전체 회원) 단위의 휴일(공휴일 및 팀원 개별 휴가)을 생성, 수정, 삭제하고, 이를 프로젝트 간트 차트에 시각적으로 표시하여 일정 관리를 용이하게 합니다.

## 사용자 시나리오
1. 사용자가 랩/워크스페이스 단위의 '휴일 관리' 메뉴에 진입하여 목록을 확인합니다.
2. 사용자가 새로운 휴일(이름, 시작일, 종료일, 비고, 타입-공휴일/휴가)을 등록합니다.
   * 휴가일 경우 특정 팀원을 선택하여 연결합니다.
3. 시스템이 `holidays` (또는 `holiydays`) 테이블에 데이터를 저장합니다.
4. 사용자가 특정 프로젝트의 일정 현황로 이동하여 '일(Day)' 보기 모드를 선택합니다.
5. 간트 차트 타임라인 영역에 등록된 휴일 구역이 시각적으로 표시되는 것을 확인합니다.

## DB 변경사항
- 파악된 내용에 따르면 Supabase에 `holidays` (또는 `holiydays`) 테이블이 만들어져 있다고 가정합니다.
- 테이블 필수 모델/컬럼 구조 검증 (필요시 마이그레이션 변경):
  - `id` (uuid, pk)
  - `name` (text) : 휴일 이름
  - `start_date` (date) : 시작일
  - `end_date` (date) : 종료일
  - `type` (text) : 'public_holiday' (공휴일) / 'member_leave' (팀원 휴가)
  - `member_id` (uuid, nullable) : 팀원 휴가일 때 연결되는 `members`의 FK
  - `note` (text, nullable) : 비고
- RLS 정책: 전체 사용자(All Authenticated Users) 조회/생성/수정/삭제 가능 권한. (모든 사용자 권한)

## 구현 목록
### Pages
- `/holidays` (또는 설정화면 내 경로) : 워크스페이스 전역 휴일 관리(CRUD) 페이지 추가
- `/projects/[id]/page.tsx` : 서버 컴포넌트에서 전체 휴일 목록 데이터를 미리 Fetch하여 간트 컴포넌트로 전달

### Components
- `HolidayList.tsx` : 전체 휴일 목록을 보여주는 데이터 테이블 UI
- `HolidayDialog.tsx` : 휴일/휴가 생성을 위한 팝업 모달 폼 UI
- `GanttChart.tsx` (수정) : dhtmlx-gantt 설정을 변경하여 전달받은 휴일 데이터를 바탕으로 일(Day) 보기 상태일 때 타임라인 특정 일자에 배경색(휴일 마커 등)을 입히도록 커스텀 클래스 적용.

### API / Server Actions
- `getHolidays()`, `createHoliday()`, `updateHoliday()`, `deleteHoliday()` 서버 액션 작성.

### 재사용
- 기존 shadcn/ui 기반의 `DataTable`, `Dialog`, `Input`, `Select`, `Calendar` 폼 컴포넌트 활용 등.

## 엣지 케이스 처리
- **타입 종속성:** 휴일 종류가 '팀원 휴가'인 경우 반드시 1명 이상의 대상을 선택하도록 폼 밸리데이션 처리.
- **간트 차트 렌더링:** 휴일이 추가/삭제되더라도 기존 테스크의 일정 데이터(시작일/종료일) 수치는 변경(연기/당김)하지 않고, 오직 "일정 현황에 휴일 구역만 표시" 처리합니다.

## 완료 기준
- [ ] 휴일 데이터의 목록 렌더링, 추가, 수정, 삭제가 잘 작동하는가.
- [ ] '팀원 휴가' 타입을 선택할 때 실제 팀원을 지정할 수 있는가.
- [ ] 개별 프로젝트 일정 현황의 '일(Day)' 모드에서 등록된 전체 휴일이 타임라인 배경에 구별되어 표시되는가.
- [ ] 간트뷰에 휴일이 표시되더라도 기존 등록 업무들의 실제 DB상 시작일/종료일 값이 변형되지 않는가.
