# Feature Spec: 달력 기반 휴일관리 UI 개편

## 목적
팀원 전체가 직관적인 달력(Calendar) 화면을 통해 본인 및 다른 팀원의 휴가, 출장, 워크샵 등 다양한 일정을 한눈에 확인하고 손쉽게 등록·수정·삭제할 수 있게 개선한다. 기존의 검색 위주 목록형 UI는 '프로젝트 관리' 화면처럼 탭 형태로 분리하여 접근성을 유지한다.

## 사용자 시나리오
1. **화면 진입 및 기본 뷰**: 사용자가 `휴일관리` 메뉴에 진입하면 '달력 뷰'가 기본 탭으로 표시되며, 이번 달의 모든 팀원 일정이 유형별 식별 색상 뱃지(휴가: 파랑, 공휴일: 빨강 등)로 나타난다.
2. **연/월 이동 및 검색**: 상단 화살표(`<`, `>`), `오늘` 버튼 및 DatePicker를 통해 원하는 달이나 당일로 즉시 이동한다. 또한 상단 필터를 통해 특정 팀원의 일정만 모아 볼 수 있다.
3. **직관적인 식별**: 오늘 날짜의 배경은 연한 파란색으로, 휴일(공휴일 등)의 배경은 연한 빨간색으로 채색되어 주요 날짜를 한눈에 파악한다.
4. **일정 조회 및 요약**: 하루에 여러 사람의 일정이 겹쳐 공간이 부족하면 2~3개만 표시되고 `+N개 더보기` 라벨이 나타난다. 라벨을 클릭하거나 마우스를 올리면 해당 일자의 전체 일정이 팝오버(Popover)로 정리되어 보인다.
5. **일정 편집 (수정/이동)**: 기존 일정을 드래그 앤 드롭으로 다른 날짜로 손쉽게 이동하거나 넓혀서 기간을 늘린다. 일정을 직접 클릭하면 기존 목록뷰에서 쓰던 <수정 모달 컴포넌트>가 재사용되어 상세 내용을 즉시 수정/삭제할 수 있다.
6. **일정 등록**: 달력의 빈칸을 클릭하거나 특정 기간을 드래그해 놓으면 해당 기간이 자동 입력된 <등록 모달 컴포넌트>가 열리며 빠르게 완료된다.
7. **기존 목록형 UI 제공**: 검색과 리스트 확인이 편한 기존 UI는 '목록 뷰' 탭을 누르면 언제든지 기존 그대로 사용할 수 있다.

## DB 변경사항
- 테이블: `holidays`
- 변경 대상: `type` (또는 해당 컬럼의 제약 조건 업데이트)
  - 기존 유형: `'public_holiday'`, `'member_leave'`
  - 추가 유형: `'business_trip'`(출장), `'workshop'`(워크샵), `'other'`(기타)
  - *모든 추가 유형은 간트차트 시스템에서 '휴일(비업무일)'과 동일한 취급을 받아 작업 기간 산정 로직에서 제외된다.*

### Pages
- **[/holidays/page.tsx](file:///d:/develop/workspace/taskaio/src/app/holidays/page.tsx)**: 전체 컨테이너를 shadcn `Tabs` 컴포넌트로 감싸고 '달력 뷰'와 '목록 뷰' 두 개의 탭 인디케이터 제공.

- **`HolidayTabs` (신규)**: 클라이언트 환경에서 달력 뷰와 목록 뷰 간의 상태 전환 관리.

  - 연/월 Datepicker 및 `[오늘] < >` 네비게이션 제어 바
  - 팀원별 정보 필터링 Select
  - 일자별 Drag & Drop 등록 및 일정 바 이동 이벤트 (HTML5 Drag Event 처리)

  - Select 항목에 '출장', '워크샵', '기타'를 추가.
  - 달력에서 초기 날짜값을 전달받아 렌더링하도록 Props 연동 강화.

## 엣지 케이스 처리
- **일정 드래그 시 검증 (Validation)**: 이미 존재하는 본인의 타 일정과의 날짜 중복 시 어떻게 처리할 것인가? 백엔드 `error` 또는 훅에서 검출 후 낙관적 롤백 처리 진행.
- **다수 일정 렌더링**: 모바일 환경이나 너비가 좁은 화면에서 일정 텍스트가 삐져나오지 않도록 Flex-truncate 적용, 최대 2개만 렌더링 후 나머지는 Popover 처리 규칙 통일.
- **시작/종료일 역전**: 드래그 앤 드롭 중 종료일이 시작일보다 앞서게 놓였을 땐 자동으로 시작-종료를 교환(Swap)하여 올바른 날짜 폼으로 렌더링한다.
