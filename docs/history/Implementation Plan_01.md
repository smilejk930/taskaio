# WBS 일정 관리 앱 (taskaio) 초기 구축 계획

사용자 요구사항을 바탕으로 WBS 기반 일정 관리 앱의 핵심 인프라와 기능을 구축합니다.

## User Review Required

> [!IMPORTANT]
> **계층 구조 제약**: 업무 계층은 최대 2단계(상위-하위)로 제한합니다. (상위 생성 후 하위는 다수 선택 생성 가능)
> **동적 계산**: 상위 작업의 시작/종료일은 하위 작업들의 최소 시작일과 최대 종료일로, **진행률은 하위 작업들의 평균(또는 가중치 평균) 퍼센티지**로 자동 계산되어 실시간 동기화됩니다.

## Proposed Changes

### Database (Supabase)
프로젝트와 업무 관리를 위한 핵심 테이블을 생성합니다. (`supabase-mcp-server` 사용)

#### [NEW] `00_schema.sql` (마이그레이션용)
- `profiles`: 사용자 기본 정보 (Auth 연동)
- `projects`: 프로젝트 기본 정보
- `project_members`: 프로젝트별 팀원 및 권한 (`owner`, `manager`, `member`)
- `tasks`: 업무 정보 (계층 구조, 일정, 담당자, 상세 내용 등)
- `holidays`: 공휴일 및 팀원 개인 휴가 일정 테이블
- RLS 정책: 프로젝트 멤버만 해당 프로젝트 및 관련 휴일 데이터를 조회 가능하도록 설정

---

### Core Components
공통으로 사용될 UI 및 상태 관리 로직을 구현합니다.

#### [NEW] `src/hooks/use-tasks.ts`
- 프로젝트별 업무 목록 조회 및 실시간 업데이트 훅
#### [NEW] `src/store/use-project-store.ts`
- 현재 선택된 프로젝트 및 필터링 상태 관리 (Zustand)

### Pages & UI
핵심 화면 3곳을 단계적으로 고도화합니다.

#### [NEW] `src/app/projects/page.tsx`
- 프로젝트 목록 조회 및 새 프로젝트 생성 모달

#### [NEW] `src/app/projects/[id]/page.tsx` (Phase 2)
- **GanttChart 고도화**:
  - 일/주/월 단위 전환 및 담당자/상위업무 필터링
  - 휴일(공휴일/개인휴무) 실시간 마커 표시
  - 업무 드래그 시 DB 즉시 업데이트 액션 연동
- **WBS 그리드(WbsGrid)**:
  - TanStack Table 기반 엑셀 스타일 실시간 편집
  - 행 추가(하위 업무 생성) 및 삭제 기능
- **휴일 관리(HolidayManager)**:
  - 프로젝트별 휴일 등록 및 관리 다이얼로그
- **프로젝트 대시보드(ProjectSummary)**:
  - 상단 요약 카드 (전체 진행률, 잔여 업무, 팀원 현황)

## Verification Plan

### Automated Tests
- Supabase SQL 유닛 테스트 (pgTAP 또는 직접 쿼리 검증)
- 업무 계층 구조 밸리데이션 로직 테스트

### Manual Verification
1. **프로젝트 생성**: 새로운 프로젝트를 생성하고 목록에서 확인
2. **업무 추가 및 상세 입력**: 상위 업무 생성 후 상세 내용을 입력하고, 하위 업무를 여러 개 추가
3. **일정 및 진척률 자동 계산**: 하위 업무의 날짜나 진척률(%)을 변경했을 때 상위 업무의 기간과 전체 진행률이 자동으로 업데이트되는지 확인
4. **간트 필터 테스트**: 상위 업무만 보기, 특정 팀원 업무만 보기 등 필터 기능이 정확히 작동하는지 확인
5. **드래그앤드롭**: 간트 차트 빈 공간을 드래그하여 신규 업무를 등록하고, 기존 바를 드래그하여 수정한 결과가 DB에 반영되는지 확인
