# Walkthrough: 프로젝트/팀원별 색상 지정 및 업무 연동 정규화

프로젝트 내 팀원별 고유 색상이 제대로 표시되지 않고, 업무 등록 시 색상이 자동 연동되지 않는 문제를 해결하였습니다.

## 구현된 파일
- **수정**: `src/app/projects/[id]/page.tsx`
    - 서버에서 데이터를 가져올 때 `colorCode` 필드를 누락했던 부분을 수정하였습니다.
- **수정**: `src/components/projects/ProjectClientView.tsx`
    - 업무 생성 다이얼로그를 열 때, 하드코딩된 랜덤 색상이 아닌 현재 로그인 유저의 지정된 색상을 기본값으로 사용하도록 변경하였습니다.
- **수정**: `src/components/projects/TaskDialog.tsx`
    - 다이얼로그 초기 로드 시, 담당자가 지정되어 있지만 색상 정보가 없는 경우 담당자의 색상을 DB에서 찾아 자동으로 매칭하는 로직을 추가하였습니다.

## 완료 기준 체크
- [x] 팀원 관리 화면에서 모든 팀원의 색상이 DB 값과 일치하게 표시됨
- [x] 팀원 추가 시 고유한 랜덤 색상이 데이터베이스에 저장됨 (기존 로직 정상 작동 확인)
- [x] 새 업무 등록창 오픈 시 본인의 색상이 기본값으로 선택됨
- [x] 담당자 변경 시 업무 색상이 해당 담당자의 색상으로 자동 변경됨

## 테스트 결과
- 서버 사이드 매핑 수정으로 인해 `TeamManagementView`와 `TaskDialog` 모두에서 `members` 데이터가 `colorCode`를 정상적으로 포함하고 있음을 확인하였습니다.
- 새로운 업무 등록 및 담당자 변경 시 색상이 즉각적으로 동기화되는 것을 확인하였습니다.

## 다음 단계 제안 (선택)
- 현재는 프로젝트 멤버별 고용한 색상을 사용하고 있지만, 나중에 프로필 레벨(전역) 색상 설정을 추가하여 모든 프로젝트에 일관된 색상을 적용하는 기능을 검토할 수 있습니다.
