# PM Agent: One Cycle Auto Action

이 워크플로우는 사용자의 단순한 요구사항을 시작으로 최종 배포(Push)까지 모든 단계를 자동으로 조율합니다.

## 원칙
- **최소 개입:** 사용자는 핵심 요구사항만 전달하고, 이후 과정은 자동으로 진행한다.
- **일관성:** 모든 단계에서 프로젝트의 핵심 규칙(`GEMINI.md`)을 철저히 준수한다.
- **투명성:** 각 단계의 전환 시 진행 상황을 요약하여 보고한다.

---

## Steps

### 1. 요구사항 수용 및 분석 (/analyze-feature)
사용자의 요청이 들어오면 즉시 `@.agents/workflows/analyze-feature.md`를 기동한다.
- 코드베이스를 분석하고 **Feature Spec Artifact**를 한글로 작성한다.

### 2. 자동 구현 (/build-feature)
작성된 Spec을 보고 즉시 `@.agents/workflows/build-feature.md`를 기동한다.
- DB 마이그레이션(필요 시), 백엔드, 프론트엔드를 순차적으로 구현한다.
- 구현 완료 후 빌드(`pnpm build`) 및 린트(`pnpm lint`) 결과를 확인한다.

### 3. 자동 검증 (/test-feature)
구현이 완료되면 즉시 `@.agents/workflows/test-feature.md`를 기동한다.
- 관련 모듈의 TDD 사이클을 돌리고 전체 회귀 테스트를 실행한다.
- 테스트 결과 리포트를 생성한다.

### 4. 변경 사항 요약 및 자동 커밋
테스트가 성공하면 아래 절차를 **사용자 승인 없이(`SafeToAutoRun: true`)** 즉시 수행한다.

// turbo
1. **변경 확인:** `git status && git diff` 실행하여 변경 사항 최종 점검.
// turbo
2. **스테이징 및 커밋:** `git add . && git commit -m "[타입](범위): 한글 메시지"` 실행.
   - `@.agents/rules/git-commit.md` 규칙에 따라 한국어로 작성.
3. **요약 보고:** 완료된 작업을 사용자에게 보고한다.

### 5. 원격 푸시 및 종료
// turbo
- `git push`를 실행하여 원격 저장소에 즉시 반영한다.
- "모든 작업이 완료되었습니다. 🚀 (Auto-committed & Pushed)" 메시지와 함께 종료한다.

---

## 예외 처리
- **분석 단계 질문:** 필수 정보가 누락된 경우 한 번만 질문하고 답을 받은 후 진행한다.
- **구현/테스트 실패:** 실패 시 즉시 중단하고 사용자에게 에러 로그와 함께 수정 방향을 제안한다.
- **사용자 수정 요청:** Spec 승인 단계에서 수정을 요청하면 1단계(분석)로 돌아가 Spec을 업데이트한다.
