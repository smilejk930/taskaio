# Deploy

## Steps

1. **배포 전 코드 리뷰**
   `/review` 워크플로우를 호출하여 코드 품질을 검사한다. 🔴 심각도 이슈가 남아있으면 배포를 중단하고 사용자에게 확인을 요청한다.

2. **환경변수 확인**
   프로덕션 환경에서 필요한 환경변수가 모두 설정되어 있는지 확인한다.
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - `SUPABASE_SERVICE_ROLE_KEY` (서버 전용)

3. **프로덕션 빌드 확인**
   ```bash
   pnpm build
   ```
   빌드 에러가 없는지 확인한다.

4. **DB 마이그레이션 적용**
   적용되지 않은 마이그레이션 파일이 있는지 확인하고, 있다면 프로덕션 DB에 적용한다. DB 변경이 있는 경우 사용자에게 반드시 확인을 요청한다.

5. **배포 실행**
   Vercel 또는 사용 중인 배포 플랫폼에 배포한다.
   ```bash
   # Vercel CLI 사용 시
   npx vercel --prod
   ```

6. **배포 후 검증**
   브라우저 에이전트로 프로덕션 URL에 접속하여 주요 기능(로그인, 업무 생성, 간트차트)이 정상 동작하는지 확인한다.(스크린샷 Artifact를 생성❌)
