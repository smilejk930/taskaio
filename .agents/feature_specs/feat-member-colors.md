# Feature Spec: 사용자/프로젝트별 업무 색상 지정 기능

## 목적
사용자별, 프로젝트별로 고유한 색상 코드를 저장 및 지정하여, 간트차트에서 팀원별 업무 흐름(연관성)을 색상만으로 직관적으로 판별할 수 있게 합니다.

## 사용자 시나리오
1. 시스템이 새로운 팀원을 프로젝트에 초대할 때, 임의의 기본 색상을 자동으로 지정합니다.
2. 각 프로젝트 팀원은 **팀원 관리** 메뉴에서 자신의 색상(프로젝트 한정)을 선호하는 색상으로 직접 수정합니다. (색상 팔레트 선택 및 색상 코드 직접 입력 지원)
3. 새로운 업무를 등록시 업무 담당자를 선택하면, 지정된 팀원의 색상을 가져와 업무 색상으로 자동 반영합니다.
4. 업무 등록 또는 수정 모달에서 필요에 따라 해당 업무의 색상을 다르게 수정할 수도 있습니다.
5. 담당자가 없는 업무의 경우, 지정된 디폴트 색상(예: 기본 회색 계열) 규정에 따라 할당됩니다.

## DB 변경사항
- 기존 스키마 사용 **불가** (추가 정보 필요)
- **대상 테이블**: `project_members`
- **추가 컬럼**: `color_code: text("color_code")` (모든 DB_TYPE 호환을 위한 텍스트 타입. HEX 값 저장 목적)
> *참고*: 기존 입력된 업무 색상을 유지하기 위해 `tasks.color` 컬럼은 그대로 유지하고, 생성 시점에만 팀원의 색상값을 복사(Override) 해주는 식으로 동작합니다.

## 구현 목록
### Pages
- 신규 페이지 없음 (기존 팀원 관리 페이지 활용)

### Components
- `TeamManagementView.tsx`: 멤버 목록 테이블 수정, 로그인한 본인 행에 한해 색상을 변경할 수 있는 UI 표출.
- `MemberColorPicker.tsx` (신규): 제한된 색상 팔레트와 사용자 지정 커스텀 색상 입력을 모두 지원하는 색상 선택 컴포넌트.
- `TaskDialog.tsx` & `WbsGrid.tsx`: 담당자(Assignee)가 변경되면, `onChange` 이벤트를 통해 해당 담당자의 등록된 색상을 찾아 Color Field에 자동 바인딩. 담당자가 없으면 규정된 기본 색상(Default Color) 지정.

### Server Actions / API Routes
- `updateMemberColor` (신규): `projectId`, `userId`, `color` 를 받아 `project_members` 데이터를 업데이트. (권한: 본인만 수정 가능)
- `addMember` (수정): 팀원 추가 시 초기 `color_code`에 랜덤 HEX 문자열을 주입.

### 재사용
- 기존 `tasks` 스키마 및 쿼리 방식
- `shadcn/ui`의 `Popover`, `Input` 등을 조합해 ColorPicker 구현

## 엣지 케이스 처리
- **담당자 없는 업무**: `Unassigned` 처리가 될 경우 고정된 기본 색상(예: `#94A3B8` Slate-400)으로 처리합니다.
- **기존 데이터 호환**: 기존에 이미 등록된 `tasks` 데이터는 변경하지 않으며, 새로 등록/수정하는 시점에만 로직이 개입합니다. 기존 팀원들의 추가된 `color_code`는 DB 마이그레이션이 발생할 때 초기 랜덤값 또는 null 대응(클라이언트에서 null일 시 랜덤 표시 후 저장)을 진행합니다.
