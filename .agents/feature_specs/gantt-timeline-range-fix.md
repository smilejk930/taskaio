# Feature Spec: 간트차트 필터링 시 시간축 범위 동적 조절 (Gantt Dynamic Scale Fit)

## 목적
이전 설계안(프로젝트 전체 기간 유지)을 폐기하고, 사용자의 요구사항에 맞추어 **업무 변동 또는 필터링 시 해당 화면에 남은 업무들의 실제 기간에 딱 맞게 간트차트 눈금(가로축) 범위가 동적으로 갱신되도록** 수정합니다.

## 사용자 시나리오
1. 사용자가 간트차트 화면에 진입하면 현재 나열된 업무 중에서 가장 빠른 시작일과 가장 늦은 종료일을 계산해, 이에 딱 맞는 타임라인(앞뒤 약 일주일 정도 여백)을 렌더링한다.
2. 사용자가 담당자를 '김성현'으로 검색/필터링한다.
3. '김성현'의 업무가 4월 1일부터 4월 3일에 포진해 있다면, 간트차트의 가로축은 자동으로 3월 말부터 4월 초에 맞춰 조절된다.
4. 즉, 표시될 업무의 최소~최대 일정에 정확히 맞춰 버그 없이 타임라인이 축소 및 확장된다.

## DB 변경사항
- 없음

## 구현 목록
### Components
- **`GanttChart.tsx`**:
  - `ganttTasks` (필터링된 상태로 props로 들어오는 배열) 정보를 기반으로 컴포넌트 내부에서 동적 `start_date`와 `end_date`를 연산합니다.
  - 연산 로직: 
    - `tasks` 순회 -> 유효한 `start_date`, `end_date` 목록 수집
    - `Math.min`, `Math.max`로 최소 시작일 및 최대 종료일 도출
    - 약간의 여유(Margin) 확보를 위해 `startDate - 7일`, `endDate + 7일` (또는 지정일) 적용
  - 계산된 `start_date`와 `end_date`를 `ganttInstance.config.start_date`와 `end_date`에 덮어씌운 뒤, `gantt.render()` 및 `gantt.parse()`를 실행하여 강제로 스케일 범위를 재설정합니다.
  - 현재 `fit_tasks = false` 옵션으로 인해 dhtmlx가 타임라인 범위를 제대로 갱신하지 못하는 문제를 명시적 날짜 바인딩으로 해결합니다.

## 엣지 케이스 처리
- **화면에 표시될 업무가 전혀 없는 경우 (검색결과 0건 등)**: 기준이 될 날짜가 없으므로 '오늘' 기준으로 앞뒤 15일 정도의 기본 타임라인을 제공하여 빈 화면이 깨지지 않게 방어합니다.
- **업무가 단일 날짜(시작/종료일 동일)인 경우**: 최소 날짜와 최대 날짜가 동일하더라도 에러 없이 양 옆 마진이 들어가 정상적인 타임라인 크기가 확보되도록 합니다.

## 완료 기준
- [ ] 특정 조건 필터 적용 시 렌더링되지 않았던 월(예: 4월)이 업무 범위에 포함될 경우 타임라인이 해당 일자까지 정상적으로 확장/세팅되어 노출된다.
- [ ] 필터링 조건 변경으로 눈에 보이는 데이터 기간이 변경될 때마다 간트차트의 표시 영역이 해당 업무에 적합한 범위로 즉시 반응하며 축소 및 확장된다.
