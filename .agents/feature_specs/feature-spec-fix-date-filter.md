# Feature Spec: 업무 기간 조회 필터링 오류 수정

## 목적
업무 기간 조회 필터 사용 시, 시작일과 종료일이 동일한 범위(하루)를 선택하더라도 해당 날짜에 포함된 업무가 정상적으로 조회되도록 필터링 로직을 수정합니다. 현재는 시간대(Timezone) 차이 및 시간 정보 누락으로 인해 동일 날짜 업무가 검색에서 제외되는 버그가 있습니다.

## 사용자 시나리오
1. 사용자가 업무 목록(WBS) 또는 일정 현황(Gantt) 탭에서 기간 필터를 클릭합니다.
2. 특정일(예: 2026-03-28)을 시작일과 종료일로 동일하게 선택합니다.
3. 해당 날짜(2026-03-28)에 걸쳐 있는 업무가 목록에 정상적으로 표시되는지 확인합니다.

## 코드베이스 분석 결과
- **관련 파일**: `src/hooks/use-task-filters.ts`
- **버그 원인**: 
  - `new Date(task.start_date)`는 문자열 `YYYY-MM-DD`를 UTC 기준으로 파싱합니다 (예: 2026-03-28 09:00:00 KST).
  - 반면, 캘린더에서 선택된 `filters.dateRange.to`는 로컬 시간 기준 00:00:00입니다 (예: 2026-03-28 00:00:00 KST).
  - 비교 식 `taskStart > filters.dateRange.to`가 `09:00:00 > 00:00:00`으로 평가되어 참(True)이 되고, 결국 해당 항목이 필터링(`return false`)되어 버립니다.

## DB 변경사항
- 없음 (기존 `tasks` 테이블의 `startDate`, `endDate` 컬럼 활용)

## 구현 계획
### Hooks
- `src/hooks/use-task-filters.ts`:
  - 날짜 비교 로직을 `Date` 객체 직접 비교 대신 `YYYY-MM-DD` 문자열 비교 방식으로 변경합니다.
  - 이를 통해 시간대 전송이나 파싱 방식의 차이에 관계없이 정확한 날짜 기반 매칭을 보장합니다.

### Components
- `src/components/projects/TaskSearchFilter.tsx`: 기존 `Date` 객체 기반 필터 상태 유지는 하되, `useTaskFilters` 내부에서 비교 시에만 변환하여 처리합니다.

## 엣지 케이스 처리
- **날짜 정보 누락**: `start_date`나 `end_date`가 없는 업무는 기간 필터 적용 시 제외 처리(현재 로직 유지).
- **역전된 범위**: 종료일이 시작일보다 빠른 경우 필터링 결과 없음(정상).
