# Feature Spec: 간트차트 횡 스크롤 복구 및 기간 표시 개선

## 목적
간트차트의 타임라인(기간) 영역에 가로 스크롤(횡 스크롤)이 생성되지 않아 일정이 긴 경우 각 열(컬럼)이 컨테이너에 맞춰 강제 축소되며 글자나 기간이 제대로 표시되지 않는 문제를 해결한다.

## 사용자 시나리오
1. 사용자가 기간이 길거나 여러 달에 걸친 업무가 등록된 간트차트 뷰로 이동한다.
2. 시스템이 간트차트를 렌더링할 때, 타임라인의 각 열(Day/Week/Month)이 최소 너비를 유지하고 화면 너비를 초과할 경우 하단에 가로 스크롤바를 자동으로 생성한다.
3. 사용자가 가로 스크롤을 이용해 전체 기간의 간트 그리드를 끊김 없이 정상적인 비율로 확인할 수 있다.

## DB 변경사항
- **없음** (프론트엔드 라이브러리 설정 수정 단독 이슈)

## 구현 목록
### Components
- `GanttChart` (`src/components/gantt/GanttChart.tsx`):
  - `ganttInstance.config.autofit = false`를 명시적으로 추가하여 타임라인 컬럼이 좁은 영역에 억눌리지 않고 고유 너비를 가지도록 수정.
  - `day` 스케일 뿐만 아니라 `week`, `month` 스케일에도 `min_column_width`를 적용(예: day=70, week=90, month=120)하여 모든 뷰에서 횡 스크롤이 작동하도록 유도.
  - 업무의 가장 빠른 날짜와 늦은 날짜를 기준으로 양 끝 여백을 두어(start_date, end_date padding) 드래그 및 일정 확인이 용이하도록 여유 구간 확보를 고려.

### 재사용
- 기존 `dhtmlx-gantt` 인스턴스 초기화 및 스케일 업데이트 로직(`useEffect`)을 재사용 및 확장.

## 엣지 단말 케이스 처리
- **일정이 매우 짧거나 없을 경우**: `min_column_width`에 의해 빈 그리드 뷰가 확보되므로 어색하게 타임라인이 끊어지지 않도록 함.
- **화면 사이즈 변경 시(Responsive)**: 브라우저 크기 변경에도 컬럼이 찌그러지지 않고 대신 스크롤바 영역 크기만 동적으로 변하게 됨.

## 완료 기준
- [ ] 간트차트 타임라인 하단에 정상적인 가로 스크롤이 노출되는가
- [ ] 화면 사이즈를 줄였을 때 타임라인 컬럼이 찌그러지지 않고 `min_column_width`를 유지하는가
- [ ] 일(Day) / 주(Week) / 월(Month) 스케일로 각각 변경해도 모두 횡 스크롤이 정상 동작하는가
