# Feature Spec: WBS 그리드 업무명/설명 UI 개선

## 목적
WBS(Work Breakdown Structure) 그리드의 업무 목록 탭에서 업무명과 설명 칸의 다중 행(엔터) 입력을 제대로 지원하고, 읽기 모드에서도 줄바꿈이 반영되도록 표시하며, 입력/조회 시 높이 문제를 개선한다.

## 사용자 시나리오
1. 사용자가 WBS 그리드에서 여러 줄로 구성된 업무명이나 설명을 입력한다.
2. `Textarea`의 입력 창이 다중 행을 입력하기에 충분한 기본 크기를 제공하여 편리하게 편집할 수 있다.
3. 저장을 완료한 후 조회(읽기) 모드에서도 단순 한 줄이 아니라, 사용자가 입력한 엔터(줄바꿈)가 그대로(`white-space: pre-wrap`) 유지되어 표시된다.

## DB 변경사항
- 없음 (프론트엔드 UI 수정)

## 구현 목록
### Components
- `WbsGrid` (`src/components/wbs/WbsGrid.tsx`):
  - **업무명(`title`) 편집/조회 모드:**
    - 편집 모드: 기존 `Input`을 `Textarea`로 교체하고, 다중 행 입력에 적합하도록 최소 높이(`min-h-[60px]` 이상) 부여.
    - 조회 모드: 기존의 단순 텍스트 표시(span) 영역을 멀티라인 표시에 적합하게(`whitespace-pre-wrap`) 변경.
  - **설명(`description`) 편집/조회 모드:**
    - 편집 모드: 기존 `Textarea`에 고정되어 있던 `h-8` 및 `resize-none` 클래스를 제거하고, 동적 크기를 갖거나 충분한 기본 높이(`min-h-[60px]`)를 갖도록 수정.
    - 조회 모드: 기존 `line-clamp-1` 클래스를 제거하고 `whitespace-pre-wrap`을 적용하여 사용자가 입력한 줄바꿈이 전부 보여지게 설정.

### 재사용
- 기존 `WbsGrid`의 구조 변경 없이 TanStack Table 컬럼 정의 부분(cell 렌더러)만 수정.

## 엣지 케이스 처리
- **엔터키(Enter) 이벤트 처리:** 기존 `Input`에서는 Enter를 누르면 `handleSave`가 동작했음. `Textarea`에서는 순수 Enter는 줄바꿈 동작이 되어야 하므로, 저장은 별도 액션 혹은 `Ctrl+Enter` / `Shift+Enter`로 처리하거나, 기존 행 우측의 `Save` 버튼을 이용하는 것으로 유도. 
  *(이번 변경에서는 업무 단축키 충돌을 막기 위해 `Textarea`에서의 `Enter` 저장을 비활성화합니다)*
- **행 높이의 불규칙 문제:** 멀티라인 데이터가 들어오면 해당 행의 높이만 증가하게 되는데, 그리드의 특성상 가독성을 해치지 않게 패딩을 유지.

## 완료 기준
- [ ] 업무명과 설명이 모두 텍스트 영역(`Textarea`)를 통해 입력 가능한가
- [ ] 입력 창이 1줄(h-8)로 눌려 있지 않고 적절한 높이를 가지는가
- [ ] 수정 후 목록에서 엔터처리가 정상적으로 유지된 상태로 데이터가 표시되는가
